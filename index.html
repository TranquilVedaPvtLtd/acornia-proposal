The RM reviews and approves everything before tasks are created. The processing cost (~₹50-100 per meeting) is billed at actuals on Acorn Wealth's own OpenAI account — we don't mark it up.

We have no interest in holding your data or operations hostage — that's the Zoho model, not ours.

Phase Deliverables & Acceptance Criteria

Phase 1 — Core Platform Foundation
Weeks 1–4
Deliverables
  • Supabase PostgreSQL schema: 16 tables (Investor, Task, Prospect, Activity, Portfolio_Review, Meeting_Notes, Document, Folio, Notification, Audit_Log, Api_Settings, Custom_Role, Holiday, Template_Step, Platform_Credentials, Transaction_Draft)
  • Row-level security (RLS) policies per role: Super Admin, Admin, RM, Ops
  • FastAPI backend: JWT auth, password hashing, role middleware, rate limiting
  • Next.js 14 App Router: login, dashboard shell, sidebar navigation, role-based route guards
  • Investor management: list (search, filter, sort), detail 360 view, create/edit form, bulk CSV import
  • CVL KRA KYC API: real-time KYC status verification by PAN
  • Zoho data migration script: investor records exported from Zoho → imported to PostgreSQL
  • Hostinger VPS: Docker Compose (frontend + backend + db), SSL/HTTPS, CI/CD via GitHub Actions
Acceptance Criteria

Investor list: 500+ investors load in <2 seconds. Search returns results in <1 second. PAN uniqueness enforced at DB level.

Auth/RBAC: All 4 roles log in successfully. RM cannot access Admin-only routes — verified by manual role-switching test.

CVL KRA: KYC status check returns result in <5 seconds for a valid PAN. Invalid PAN returns appropriate error.

Migration: Record count in PostgreSQL matches Zoho export count exactly. Zero data loss on all required fields.

CI/CD: Push to main branch triggers deployment, live on VPS within 5 minutes.

Phase 2 — Transactions + Dashboard
Weeks 5–9
Deliverables
  • Task management: 7 transaction types (SIP, SWP, Purchase, Redemption, Switch, STP, NFO) with type-specific checklist steps
  • Task status workflow: Draft → In Progress → Pending Approval → Completed / Rejected
  • Ops approval queue: review pending tasks, approve/reject with reason logging
  • Admin dashboard: AUM trend chart, task pipeline KPIs, team performance metrics, top investors
  • RM dashboard: my investors (assigned), open tasks, upcoming reviews, recent activities
  • Ops dashboard: task queue by type/status, overdue tasks, approval backlog count
  • Supabase Realtime: live in-app notifications (task assigned, status changed, KYC updated)
  • Prospects pipeline: kanban view (stage-based), list view, lead source tracking, RM assignment
  • Activities module: log calls/meetings/emails/notes per investor, activity timeline view
  • Audit log: immutable record of all data changes — form, field, old/new value, user, timestamp
  • AMFI NAV daily sync: background job fetches all scheme NAVs every evening
  • Transaction history migration from Zoho
Acceptance Criteria

Task types: All 7 types create with correct checklist steps. Status transitions follow defined workflow only.

Realtime: Ops approves a task — RM dashboard updates within 3 seconds (Supabase Realtime verified).

Dashboards: Admin dashboard loads <3s for 500+ investor dataset. Chart aggregates verified against direct DB query.

AMFI sync: NAV data auto-updates at scheduled time. Last-sync timestamp visible in settings. 5,000+ scheme NAVs importable.

Audit log: Every field edit creates an audit record. Audit records cannot be deleted by any role (RLS enforced, tested by all 4 roles).
